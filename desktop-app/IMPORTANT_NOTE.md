# ⚠️ ВАЖНО: Адрес для подключения

## Используйте адрес FRONTEND, а не BACKEND!

Десктопное приложение должно обращаться к **веб-интерфейсу (frontend)**, а не к backend напрямую.

### Почему?

- Frontend проксирует запросы к backend автоматически
- Frontend обычно прокинут в сеть и доступен извне
- Backend может быть недоступен напрямую из сети

### Как узнать правильный адрес?

1. **Откройте веб-интерфейс в браузере**
2. **Посмотрите адресную строку** - это адрес frontend
3. **Добавьте `/api` в конце** - это будет ваш `API_BASE_URL`

### Примеры:

Если в браузере адрес:
- `http://192.168.1.100:3000` → используйте `http://192.168.1.100:3000/api`
- `https://ftr.lilfil.ru` → используйте `https://ftr.lilfil.ru/api`
- `http://localhost:3000` → используйте `http://localhost:3000/api`

### Настройка в .env:

```env
API_BASE_URL=http://ваш-адрес-frontend/api
```

**НЕ используйте:**
- ❌ `http://localhost:3001/api` (это backend)
- ❌ `http://192.168.1.100:3001/api` (это backend)

**Используйте:**
- ✅ `http://localhost:3000/api` (это frontend)
- ✅ `http://192.168.1.100:3000/api` (это frontend)
