# Быстрый старт

## Шаг 1: Установка зависимостей

```bash
pip3 install -r requirements.txt
```

## Шаг 2: Настройка .env файла

Скопируйте пример и отредактируйте:

```bash
cp .env.example .env
```

### Минимальная настройка:

Откройте файл `.env` и укажите адрес **ВЕБ-ИНТЕРФЕЙСА** (frontend):

```env
API_BASE_URL=http://localhost:3000/api
```

**ВАЖНО:** 
- Используйте адрес **frontend** (веб-интерфейса), а не backend напрямую!
- Frontend проксирует запросы к backend автоматически
- Если frontend на порту 3000 → `http://localhost:3000/api`
- Если frontend на другом порту → измените порт
- Если сервер удаленный → укажите адрес frontend (например, `http://your-domain.com/api` или `https://your-domain.com/api`)

### Как узнать адрес сервера?

1. **Локальный Docker сервер:**
   ```bash
   # Проверьте, запущен ли backend
   docker ps | grep backend
   # Обычно это http://localhost:3001/api
   ```

2. **Удаленный сервер:**
   - Спросите у администратора
   - Или посмотрите адрес в браузере при работе с веб-версией

3. **Проверка доступности:**
   ```bash
   curl http://localhost:3001/api/health
   ```

## Шаг 3: Запуск приложения

```bash
python3 main.py
```

## Что дальше?

1. При первом запуске появится окно входа
2. Введите ваши учетные данные (те же, что и для веб-версии)
3. После входа данные автоматически синхронизируются с сервером
4. Готово! Теперь можно работать оффлайн

## Проблемы?

### Ошибка подключения к серверу
- Проверьте, что сервер запущен
- Проверьте правильность `API_BASE_URL` в `.env`
- Проверьте интернет-соединение

### Ошибка "command not found: pip"
- Используйте `pip3` вместо `pip` на macOS/Linux
- Или `python3 -m pip install -r requirements.txt`

### Приложение не запускается
- Проверьте логи: `~/.local/share/ftr_registration/logs/app.log`
- Убедитесь, что все зависимости установлены: `pip3 list`
